1 Introduction
Quantum machine learning is the area of obtaining insights from data and phys-
ical phenomena using quantum computers and algorithms. This area of research
has been explored as early as Trugenberger [2] and developed during the 2010s
by Lloyd, Mohseni, and Rebentrost [3], Schuld and Petruccione [6], amongst
others. As classical and quantum machine learning become more relevant, there
is a need to compare new results in the ﬁeld with previous work. However, the
lack of both implementations and speciﬁcations makes such comparison diﬃ-
cult, delaying further progress.
In this project, we aim to understand and implement some of the most suc-
cessful papers in quantum machine learning as a baseline for novel development
in the ﬁeld. Also, we will provide test runs for each model and a comparison
to its most related classical counterpart, if possible. The following papers have
reported the highest accuracy in the corresponding datasets:

2 Experimental setup and datasets description
Regarding machine learning in general, there has been a tremendous eﬀort to
build labeled datasets to test classiﬁcation algorithms. Therefore, depending on
the architecture we will test the models with the following datasets.
MNIST Digits
The National Institute of Standards and Technology (NIST) collected a database
of handwritten digits by 500 diﬀerent writers. Thereafter, it was modiﬁed in Le-
Cun et al. [1] —hence the acronym Modiﬁed National Institute of Standards and
Technology database (MNIST)— for document and image recognition. Histor-
ically relevant, this database is used to test the reproduction of the model in
Farhi and Neven [4], as they showed that it was possible to learn with Noisy
Intermediate-scale Quantum devices (NISQs).

Moons dataset
The Moons dataset was generated with 1000 samples using the scikit-learn func-

2.1 Implementation
As part of the project, we provide a repository with the code, where the follow-
ing implementations can be found as Python classes and their respective note-
book companions. Following the instructions in the papers as closely as possible,
we will implement the models as ready-to-use classes in Python, speciﬁcally us-
ing Tencent’s TensorCircuit, a fast quantum computing simulation framework
based on hardware acceleration, just-in-time compilation, and automatic diﬀer-

We implemented the classical counterparts of the models according to
the instructions found in the papers or a reasonably close equivalent. For in-
stance, as stated in Farhi and Neven [4] (p. 12),
Here we run a (Matlab classical) classiﬁer with one internal layer
consisting of 10 neurons. Each neuron has 16 coeﬃcient weights and
one bias weight so there are 170 parameters on the internal layer and
4 on the output layer. The classical network has no trouble ﬁnding
weights that give less than one percent classiﬁcation error on the
training set. The (Matlab) program also looks at the generalization
error but to do so it picks a random 15 percent of the input data to
use for a test set.

4 Comments & Conclusion
Understanding and implementing state-of-the-art models requires a deep com-
prehension of the fundamentals, from matrix calculus to quantum operators.
Moreover, there is an implicit complexity in trying to reproduce the experi-
ments, as details are commonly disregarded, and admittedly, most of the time
was spent on understanding the authors’ intentions.
Regarding the implementation, originally the models were executed using
Qiskit. We recognize the ease of use for introductory topics, but the scalabil-
ity for more complex algorithms is not the best. A ﬁrst migration was made
to Xanadu’s Pennylane, a framework intended for quantum machine learning.
The main problem persisted, as the running time for model training was subop-
timal. Finally, we settled with TensorCircuit, as quantum simulation time relied
on classic techniques for optimization. Hence, we acknowledge that a reason-
able period was lost transpiling the code to three diﬀerent frameworks, with
very diﬀerent syntax and speciﬁcations.
In aim to increase the reproducibility of the models, the code was uploaded

to a public repository on GitHub. In the future, we hope to implement more
models, i. e., Schuld et al. [8], and Watkins, Chen, and Yoo [10] which have
shown excellent results.
